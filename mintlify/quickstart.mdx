## 1. Get an API key

Create an account at the dashboard:

https://tryyomira.com/admin.html

Store the returned API key as an environment variable.

```bash
export YOMIRA_API_KEY="sim_..."
```

If you are using an AI agent, you usually do not need to write curl yourself. Give the key to the agent and ask it to follow the [agent setup](/agent/setup) guide.

For the agent-native setup, include the automatic hook/rule layer:

```bash
npx -y github:ryuzo-k/yomira init --api-key sim_YOUR_KEY --target all --with-mora --hooks
```

Claude Code gets a real `UserPromptSubmit` hook. Codex and Cursor get always-available project instructions/rules so the agent proactively considers Yomira when a decision depends on human reaction.

Example ask:

```text
Use Yomira to simulate reactions before I send or publish this. Gather the missing context first, then call the real API.
```

## 2. Create a simulation

The API checks for a concrete artifact or concrete options, a decision objective, and a specific audience/channel. If those are missing, it returns `422` with `missing_context` so the agent can ask the user for the missing pieces.

```bash
curl -s -X POST "https://tryyomira.com/api/simulate" \
  -H "content-type: application/json" \
  -H "x-api-key: $YOMIRA_API_KEY" \
  -d '{
    "objective": "Decide whether to publish this landing page.",
    "artifact": {
      "type": "landing_page",
      "content": "Paste the exact page, message, post, or offer people will see."
    },
    "audience": {
      "description": "Describe who will see it, where they see it, what they already believe, and what action matters."
    },
    "simulation": {
      "mode": "fast",
      "target_n": 40,
      "max_agent_voices": 8,
      "max_output_tokens": 30000
    }
  }'
```

The API returns `202 Accepted` with a `simulation_id`.

Do not treat `202` as a failure. Yomira simulations are asynchronous.

## 3. Poll the result

```bash
curl -s "https://tryyomira.com/api/simulations/SIMULATION_ID" \
  -H "x-api-key: $YOMIRA_API_KEY"
```

The completed result includes:

- `trust_layer`: grounding level, missing context, assumptions, limits, and validation next steps.
- `audience_construction_report`: the synthetic audience segments used in the run.
- `reaction_distribution`: reaction percentages and counts.
- `voice_clusters`: grouped private voices.
- `comparison`: option-by-option summaries when you send multiple options.

## 4. Compare options

Use `options` when you need a decision between multiple concrete artifacts.

```bash
curl -s -X POST "https://tryyomira.com/api/simulate" \
  -H "content-type: application/json" \
  -H "x-api-key: $YOMIRA_API_KEY" \
  -d '{
    "objective": "Choose which message to send.",
    "audience": {
      "description": "Potential early users who liked a post about agent-native reaction simulation."
    },
    "options": [
      {
        "label": "Short ask",
        "artifact": { "type": "message", "content": "Want me to run one Yomira simulation for something you are working on?" }
      },
      {
        "label": "Long context",
        "artifact": { "type": "message", "content": "I am building Yomira, an API for agent-native reaction simulation. Want to try it and give blunt feedback?" }
      }
    ],
    "simulation": {
      "mode": "fast",
      "target_n": 40,
      "max_agent_voices": 8
    }
  }'
```

Credits are charged per option.

## 5. Log real outcomes

After you use the artifact, append the real outcome to the saved simulation. This is an outcome log for comparison; it does not automatically retrain the model in self-serve today.

```bash
curl -s -X POST "https://tryyomira.com/api/simulations/SIMULATION_ID" \
  -H "content-type: application/json" \
  -H "x-api-key: $YOMIRA_API_KEY" \
  -d '{
    "actualOutcome": "Sent to 12 people. 4 replied. 1 asked for the link.",
    "notes": "The simulation matched curiosity, but underestimated how many people asked about validation."
  }'
```

## 6. Export Markdown

```bash
curl -L "https://tryyomira.com/api/simulations/SIMULATION_ID?format=markdown" \
  -H "x-api-key: $YOMIRA_API_KEY" \
  -o simulation.md
```

## What to include

The output is only useful when the input contains real context.

Include:

- The exact artifact people will see.
- The target audience.
- The channel where they see it.
- Sender or company context.
- The desired action.
- Known worries, objections, and alternatives.
