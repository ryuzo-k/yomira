Agent Simulation API は、メッセージ、投稿、LP、オファー、プロダクト案、事業案を公開・送信・開発する前に、人間がどう受け取るかを大量にシミュレーションするためのAPIです。

課題

多くのAIは、それっぽい回答を1つ返します。でも意思決定で本当に知りたいのは、自分が出そうとしているものが、相手にどう解釈されるかです。自分では明確な文章でも、受け手には怪しい、面白い、安っぽい、信頼できる、意味が分からない、と全く違う形で受け取られます。

プロダクト

Agent Simulation API は、対象となるオーディエンスの反応をシミュレーションし、誰が興味を持つのか、誰が疑うのか、どこで誤解するのか、どんな言葉で本音を言うのかを、分布と生の声として返します。Claude Code、Codex、Cursor、Hermes Agent、OpenClaw、スクリプト、自社プロダクトから利用できます。

用途

  • コンテンツ公開前チェック — 投稿、記事、広告、GEOページを出す前に、どう受け取られるかを見る。
  • メッセージの意思決定 — 営業DM、フォローアップ、投資家向け連絡、ユーザーインタビュー依頼を送る前に比較する。
  • LP・オファー検証 — LP、価格ページ、機能説明、ローンチ文面が、買い手にどう見えるかを確認する。
  • 新規事業アイデア検証 — どこで欲しくなるのか、どこで疑われるのか、どこで支払意欲が消えるのかを見る。
  • 自社プロダクトへの組み込み — 自分たちのワークフロー、調査ツール、代理店ツール、意思決定システムにシミュレーション機能を入れる。

使い方

  1. 1. 成果物を渡す

    人に見せる文章、ページ、投稿、オファー、計画、事業案をそのまま渡します。

  2. 2. 対象者を定義する

    誰の反応を見たいのかを、広い公開対象から狭い買い手層まで指定します。

  3. 3. シミュレーションする

    APIが複数のエージェント反応を並列に生成し、分布と本音の声にまとめます。

  4. 4. 結果を使う

    JSONまたはMarkdownで出力し、次のエージェント会話、資料、意思決定に使います。

出力例

興味はあるが、まだ疑っている 38%
具体例を見ないと判断できない 24%
価値をすぐ理解した 19%
ただのプロンプトラッパーだと思った 12%
価格や導入方法が分からない 7%
「やりたいことは分かる。導入が簡単で、ただのAIっぽい感想じゃなくて本当に反応の分布が見えるなら、一回試してみたい。」

API quickstart

curl -s -X POST "https://agent-simulation-api.vercel.app/api/simulate" \
  -H "content-type: application/json" \
  -H "x-api-key: $AGENT_SIMULATION_API_KEY" \
  -d '{
    "objective": "Should we publish this landing page?",
    "artifact": { "type": "landing_page", "content": "..." },
    "audience": { "description": "founders, marketers, agencies, and AI-agent users" },
    "simulation": { "target_n": 100, "mode": "standard" }
  }'

FAQ

これはアンケートの代替ですか？

すべてのケースで代替するものではありません。まずは公開前・送信前の高速な反応チェックです。外部データを使う企業向けでは、より具体的な対象者に近づけます。

AIエージェントから使えますか？

使えます。Docs、APIキー、シミュレーションしたい成果物を渡せば、Claude Code、Codex、Cursorなどから利用できます。

結果はスコアですか？

いいえ。価値があるのは、反応の分布と、その背後にある本音の声です。

最初に誰が使うべきですか？

投稿、送信、販売、公開、ポジショニングを頻繁に行い、1つの誤解が損失になる人です。
